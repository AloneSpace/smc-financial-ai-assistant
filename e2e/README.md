# E2E Scenarios (S1–S6)

Playwright tests covering the baseline acceptance scenarios in
`docs/11_TEST_PLAN.md`. They drive the **real** streaming UI, so they need the
full stack running **and a live AI key** — grounded figures come from SQL, but
the surrounding prose is generated by the model.

## Prerequisites

1. Install the runner (not part of the workspaces):
   ```bash
   npm i -D @playwright/test
   npx playwright install chromium
   ```
2. Bring up the stack and import the data:
   ```bash
   docker compose up --build
   ./scripts/import-financial-data.sh
   ```
3. Configure a real key on the backend (`.env`):
   ```
   AI_PROVIDER=openai
   OPENAI_API_KEY=sk-...        # or ANTHROPIC_API_KEY + AI_PROVIDER=anthropic
   ```

## Run

```bash
BASE_URL=http://localhost:5173 npx playwright test
```

| Scenario | What it checks |
| --- | --- |
| S1 | SQL tool block appears; answer contains the exact DB figure `$96.99 billion` |
| S2 | Out-of-coverage year (2021) → "not available", no fabricated number |
| S3 | Stop mid-stream → button disappears, partial answer marked "Generation was stopped" |
| S4 | Usage limit → amber "Resets in …" banner, input disabled (needs low `USAGE_BUDGET_USD`, `RUN_S4=1`) |
| S5 | Hard refresh restores the conversation with no duplicate messages |
| S6 | Delete confirmation: cancel keeps, confirm removes |

> **Note:** Assertions on model prose (S2) are intentionally lenient; assertions
> on grounded figures (S1) are exact because they originate from the database.
